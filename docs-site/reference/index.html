Docs→Reference

Exact truth

Facts, flags, schemas, and nothing pretending to be a tutorial.

Reference docs should be boring in the good way: scan-friendly, exact, and not hiding key facts in narrative detours.

Reference buckets

  • CLI commands and flags
  • Configuration schema and examples
  • Environment variables
  • HTTP API endpoints
  • Runtime file layout

Good reference behavior

  • Tables when tables help
  • Copy-paste examples
  • Cross-links to guides, not repeated walkthroughs
  • One canonical place for exact values

Likely first pages

  • CLI command map
  • config.yaml schema and common settings
  • Provider configuration keys
  • HTTP API surface
  • Jobs and scheduling config

Warning

If reference pages start telling a story, they are slipping into guides again.

What this unlocks

Shorter guides, cleaner README, and fewer repeated config snippets rotting in different corners.
